# Ubiquitous Language

Canonical glossary for the Alpauls Recruitment domain. Use these terms consistently in code, conversation, and documentation.

| Term | Definition | Avoid |
|---|---|---|
| **Role** | A named set of permissions assigned to a User. Predefined roles are coded in TypeScript; the `"custom"` role carries per-user permission overrides. | Permission group, access level, user type |
| **Custom Role** | A per-user permission set stored as a JSON column (`permissions`) on the User table. When a User's role is `"custom"`, permissions are read from this column instead of the predefined role definition. Not a reusable template — each custom-user has independent permissions. | Dynamic role, flexible role, ad-hoc role |
| **Resource** | A permission domain in the access control statement (e.g., `clients`, `job_mandates`, `tasks`). Each resource defines the actions that can be granted. | Entity, domain, module |
| **Action** | A granular operation on a Resource (e.g., `create`, `read`, `assign`). Roles map Resources to lists of allowed Actions. | Permission, operation, capability |
| **Admin** | A User with the `admin` role that bypasses all permission checks. Can create, update, and archive other Users, assign roles, and manage the system. | Superuser, root, system user |
| **Task** | A trackable unit of work with ownership, deadlines, and optional links to recruitment entities. Has a Creator, an Assignee, an optional start date and a deadline. | Todo, ticket, item |
| **Task Link** | A typed reference from a Task to another domain entity. Valid entity types: `prospect`, `job_mandate`, `client`, `client_contract`. | Relation, connection, association |
| **Task Event** | An immutable record of a change made to a Task (who changed what, when). Every edit produces a Task Event. | Audit log, changelog, history entry |
| **Change Record** | An immutable record of a change made to any entity. Every core entity produces Change Records on every edit. Each entity has its own table. | Audit log, changelog, history entry |
| **Assignee** | The single User responsible for completing a Task. | Owner, responsible |
| **Creator** | The User who created a Task. Distinct from the Assignee — the Creator may assign the Task to someone else. | Author, owner, reporter |
| **Task Status** | The lifecycle stage of a Task: `todo`, `in_progress`, or `done`. | State, progress, stage |
| **Reminder** | A scheduled trigger at a concrete datetime (`trigger_at`). Either attached to a Task (with optional relative offset before the task's deadline) or standalone (absolute time only). When fired, may produce a Notification. | Alarm, alert (use **Notification** for delivered message) |
| **Notification** | A delivered message to a User, generated by system events (task assignment, status change, approaching deadline, fired Reminder, or system announcement). Has a type, title, body, and read/unread state. Distinct from Reminder. | Alert, push message, toast |
| **Supervisor** | A User who oversees one or more Subordinates. Gates access — can manage and view data scoped to their Subordinates. | Manager, boss, lead |
| **Subordinate** | A User who reports to a Supervisor. A User has at most one Supervisor. | Report, direct report, underling |
| **Client** | A company or organization that the agency recruits for. Has an Assignee (the BD responsible for the account). Created and auto-assigned to the creating BD. | Customer, company, account |
| **Client Contract** | A service agreement between the agency and a Client. Has an Assignee (initially the BD; BD may assign an RM to manage it). | Agreement, deal, contract |
| **Job Mandate** | An open position a Client wants filled. The core unit of recruitment work. Has an Assignee; assignment chain: RM creates→assigns TL→TL may re-assign to Callers. Lifecycle: open→in_progress→hired→completed (or cancelled). Multiple positions with headcounts per location stored as JSON array. | Job order, req, opening, vacancy |
| **Mandate Status** | The lifecycle stage of a Job Mandate: `open`, `in_progress`, `hired`, `completed`, or `cancelled`. | State, phase |
| **Prospect** | Any person in the system, whether actively seeking a job or not. The person entity record. Stage (prospect/applicant/candidate) is per Job Mandate. | Person, individual, lead |
| **Applicant** | A stage — a Prospect who has expressed interest in a specific Job Mandate. Not a separate entity. | Job seeker, interested candidate |
| **Candidate** | A stage — a Prospect who is QC-verified for a specific Job Mandate. Not a separate entity. | Verified prospect, qualified applicant |
| **Prospect Stage** | The lifecycle of a Prospect relative to a Job Mandate: `prospect` → `applicant` → `candidate` → `hired` (or `rejected`). QC rejection sets stage to `rejected` with reason. | Status, phase, level |
| **Prospect Mandate** | The relationship between a Prospect and a Job Mandate, storing the current stage. A Prospect may have multiple Prospect Mandate records (one per linked mandate). | Assignment, application, link |
| **Prospect Mandate Event** | An immutable record of a stage change on a Prospect Mandate (who changed what, when, on which mandate). | Stage log, transition record |
| **SLA** | A service-level agreement defining performance targets for a Client Contract or Job Mandate. Stored as a static JSON config file — not a DB entity, not access-controlled. | OLA, KPI, target, metric |

## Roles

| Code | Display Name | Scope |
|---|---|---|
| `admin` | Admin | Full access, bypasses all permission checks |
| `bd` | Business Developer | Owns client acquisition, contracts, and mandates |
| `rm` | Client Relationship Manager | Manages existing mandates; reads clients/contracts |
| `sc` | Talent Sourcer | Creates/manages prospects; assigns to mandates |
| `tl` | Team Leader | Reads mandates; creates prospects; manages tasks |
| `caller` | Caller | Creates/updates prospects; reads mandates |
| `qc` | Quality Checker | Verifies prospects on mandates; reads mandates |
| `custom` | Custom | Per-user permission overrides via `permissions` JSON column |

## Resources & Actions

| Resource | Actions |
|---|---|
| `clients` | create, read, update, archive |
| `client_contracts` | create, read, update, archive |
| `job_mandates` | create, read, update, archive, assign, link_prospect, verify |
| `prospects` | create, read, update, archive |
| `tasks` | create, read, update, archive, assign |
| `reminders` | create, read, archive |
| `team_members` | create, read, update, archive, manage-roles |
| `notification` | read, update, archive |

## Visibility

| Entity | Scope |
|---|---|
| Tasks | All authenticated users (read); update/archive by Creator/Assignee/Assignee's Supervisor |
| Prospects | All authenticated users |
| Clients, Client Contracts | Scoped by Assignee + Supervisor hierarchy |
| Job Mandates | Scoped by assignee + Supervisor hierarchy |
| Reminders | All authenticated users |
| Team Members | Admin manages; all read names |
| Notifications | Personal — user sees own only |
